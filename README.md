# Sakila SQL Practice – Advanced Interview Preparation

This repository contains my **advanced SQL practice** using the **Sakila sample database**, a widely used dataset for learning and testing relational database concepts.

The goal of this project is to **strengthen SQL skills for data analyst and data scientist interviews** by solving medium to advanced level problems that are commonly asked in real-world interviews.

---

## 📊 Dataset Overview

The **Sakila database** represents a fictional DVD rental business and includes information about:
- Customers
- Films
- Actors
- Categories
- Rentals
- Payments
- Stores
- Staff
- Locations (city, country)

It is highly normalized and ideal for practicing **complex SQL queries**.

---

## 🧠 Topics Practiced

I have practiced and implemented SQL queries covering the following key topics:

### 🔹 Core SQL
- INNER JOIN, LEFT JOIN
- Multi-table joins
- GROUP BY & HAVING
- Aggregations (`SUM`, `AVG`, `COUNT`)

### 🔹 Subqueries & Correlated Queries
- Scalar subqueries
- Correlated subqueries
- `IN` vs `NOT IN`
- `EXISTS` & `NOT EXISTS`

### 🔹 Window Functions (Advanced SQL)
- `ROW_NUMBER()`
- `RANK()` / `DENSE_RANK()`
- `LAG()` / `LEAD()`
- Running totals
- Partition-based analytics

### 🔹 Analytical & Business Queries
- Top-N analysis
- Revenue ranking
- Customer segmentation
- Category-wise performance
- Store comparison analysis

### 🔹 Date & Time Analysis
- Monthly and yearly revenue trends
- Rental gap analysis
- Customer activity over time
- Peak usage analysis
